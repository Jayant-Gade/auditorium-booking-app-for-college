## ğŸš€ Installation & Setup

### Prerequisites
- Node.js (v16 or higher)
- npm or yarn
- Expo CLI
- Android Studio (for Android development)
- Xcode (for iOS development on macOS)

### Getting Started

1. **Clone the repository**
   ```bash
   git clone https://github.com/your-username/ycce-auditorium-booking.git
   cd ycce-auditorium-booking
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Start the development server**
   ```bash
   npm start
   ```

4. **Run on device/emulator**
   - **Android**: `npm run android`
   - **iOS**: `npm run ios` (macOS only)
   - **Web**: `npm run web`

### Development Setup

1. **Install Expo CLI globally**
   ```bash
   npm install -g @expo/cli
   ```

2. **Install EAS CLI for building**
   ```bash
   npm install -g @expo/eas-cli
   ```

## ğŸ“¦ Build for Production

### Android APK/AAB

1. **Configure EAS Build**
   ```bash
   eas build:configure
   ```

2. **Build for Android**
   ```bash
   # For APK (testing)
   eas build --platform android --profile preview

   # For AAB (Play Store)
   eas build --platform android --profile production
   ```

### iOS App

1. **Build for iOS**
   ```bash
   # For simulator
   eas build --platform ios --profile preview

   # For App Store
   eas build --platform ios --profile production
   ```

## ğŸ“± Publishing to Play Store

### Step 1: Prepare for Release

1. **Update app.json**
   - Increment `version` and `android.versionCode`
   - Ensure all metadata is correct

2. **Generate App Bundle**
   ```bash
   eas build --platform android --profile production
   ```

### Step 2: Play Store Console

1. **Create Developer Account**
   - Visit Google Play Console
   - Pay one-time $25 registration fee
   - Complete account verification

2. **Create App**
   - Click "Create app" in Play Console
   - Fill in app details:
     - App name: "YCCE Auditorium Booking"
     - Default language: English (US)
     - App category: Education
     - Content rating: Everyone

3. **Upload App Bundle**
   - Go to Release > Production
   - Upload the AAB file generated by EAS Build
   - Fill in release notes

4. **App Content & Store Listing**
   - Add app description
   - Upload screenshots (phone, tablet)
   - Add app icon and feature graphic
   - Set content rating
   - Add privacy policy URL (if applicable)

### Step 3: Required Assets

Create these assets for Play Store:
- **App Icon**: 512x512 px (high-res icon)
- **Feature Graphic**: 1024x500 px
- **Screenshots**: Phone (16:9 or 9:16) and Tablet (16:10)
- **Privacy Policy**: Required for apps requesting permissions

### Step 4: Release

1. **Internal Testing** (optional)
   - Upload to Internal testing track
   - Test with limited users

2. **Production Release**
   - Upload to Production track
   - Fill all required information
   - Submit for review
